# Metadata rules

## Slug

- 3–5 words, reflecting the primary keyword
- use a hyphenated, lowercase URL format
- can be slightly longer if a shorter slug would make the topic unclear
- example: for keyword "where to buy arcade machines" → `/where-to-buy-arcade-machines`

## Title (H1 / page title)

- ideally under 55 characters
- must reflect the primary keyword
- one H1 per page — the H1 is the article title

## Meta description

- 120–145 characters
- no need to repeat the primary keyword — use relevant secondary keywords instead
- should be useful and entice the click, not just a keyword list

## Heading structure

- one H1 per page (the article title)
- logical hierarchy: H2 → H3 → H4 → H5 where needed
- headings should be meaningful and scannable, not stuffed with keywords

## FAQ blocks

- every article must include at least 5 FAQ blocks
- based on real user questions (People Also Ask, Reddit, alsoasked.com)
- FAQs should expand topic coverage and include keyword variations naturally
- FAQs should address real user uncertainties or objections
